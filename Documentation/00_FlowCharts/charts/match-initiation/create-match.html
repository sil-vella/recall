Create match

Invite-based flow: create-match-session → user search → invite-player (human: notification + pending; computer: no notification, auto-accepted in session). Create button is enabled only when 3 invited and all have status accepted. Then create_room is emitted with accepted_players (user_id, username, is_comp_player) and auto_start: true. Backend stores accepted_players on the room; match starts when effective max is reached (at create or on join). Coordinator uses the accepted_players comp list instead of fetching from the API. See Random join for the other path (no invites, timer or max players, coordinator fetches comp from Flask).

  • Effective max: Computer players do not join the WebSocket room. effectiveMax = maxSize - compCount (from accepted_players with is_comp_player: true). Auto-start when currentSize >= effectiveMax (e.g. max 4, 3 comp → start when 1 human in room).
  • Start at create: After creator auto-join, if autoStart and currentSize >= effectiveMax, _startMatchForRoom is called immediately (e.g. 1 human + 3 comp).
  • Start on join: When a second human joins, _handleJoinRoom checks the same condition and calls _startMatchForRoom when effective max is reached.
  • Coordinator: If accepted_players is in start_match data, comp players are taken from that list (no API fetch). Otherwise comp players are fetched from the Flask backend (random-join path).
  • Invite filtering: Backend checks is_comp_player; human → dutch_invite notification; computer → add to session as accepted, no notification.
