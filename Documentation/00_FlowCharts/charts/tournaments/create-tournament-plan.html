Create tournament plan

From lobby Tournaments tab: existing = only public. Create new: auto private, format (e.g. F1, F2, F3), invite players. User search filters out creator. Depending on format, creator is either auto-added as accepted (player) or not included (admin only). Number of invites depends on format. Python backend creates tournament in DB. When required number have accepted, Create and invite: creator wires into existing create_room (passing tournament data), goes to play screen to wait; this creates a single match for the tournament. Then auto notification to all HUMAN players to join; others use existing join_room. Create/join event schema includes is_tournament (bool) and tournament_data (id, name, format, leaderboard, matches_left). Then wire into existing match logic; game state gets tournament keys (name, format, players, etc).
